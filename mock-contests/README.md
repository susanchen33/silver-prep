## Overview

Mock contests are essential for USACO preparation. They help you:
- Practice time management (4 hours)
- Build mental stamina
- Identify weak areas
- Get comfortable with contest pressure

---

## How to Run a Mock Contest

### Before the Contest
1. **Set aside 4 uninterrupted hours**
2. **Prepare your environment:**
   - Close unnecessary tabs/apps
   - Have your template ready
   - Prepare snacks/water
3. **Choose a past contest** you haven't seen

### During the Contest
1. **First 10-15 minutes:** Read ALL problems
2. **Identify easiest problem:** Start there
3. **Don't get stuck:** If stuck >30 min, try another problem
4. **Submit partial solutions:** Even brute force can get 30-50%
5. **Test edge cases:** Empty input, N=1, max values

### After the Contest
1. **Upsolve:** Solve problems you couldn't finish
2. **Read editorials:** Understand optimal approaches
3. **Identify patterns:** What topics were tested?
4. **Update progress tracker**

## Common Mistakes in Contests

1. **Not reading carefully**
   - Miss constraints (1-indexed vs 0-indexed)
   - Miss output format requirements

2. **Integer overflow**
   - Always check if `long long` is needed
   - Product of two ints ‚Üí use `1LL * a * b`

3. **Array bounds**
   - Off-by-one errors
   - Not allocating enough space

4. **Forgetting file I/O**
   - USACO requires file input/output!
   - `freopen("problem.in", "r", stdin);`

5. **Not testing**
   - Always test sample cases
   - Create edge case tests
